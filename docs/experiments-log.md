# @mira Experiment Log — automated PDCA

> Generated by **mira-harness** itself: Claude drives @mira, reads the full settled
> reply, and records what worked / didn't / surprised. This is the data backing the
> [Mira AI Track feedback](mira-feedback.md) (judging = ① usefulness of feedback
> ② uniqueness of use case). The unique angle: **one AI (Claude) automatically
> experiments on another AI (@mira)**, hands-free.

## How each run is recorded

`mira-harness send` / `loop` captures the **full** settled reply — every message, edits,
**buttons** (incl. `web_app`/startapp + callback), **links** (`text_url`), and **media** —
not just the first text line. Raw machine log: a JSONL run log (`mira-runs.jsonl`); one
curated row per probe below.
